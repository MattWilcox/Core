Styleguide

This page is for the use of developers and designers. It provides a library of examples for all HTML and CSS used throughout the site. Be sure to view the source of this page for additional information.

If ever you add or change something on the site, make sure it is reflected here.

General settings

Developers

  • Indentation uses single tabs, not spaces. Indent properly. Everywhere. Always. See why.
  • We’re using the HTML5 DOCTYPE but no HTML5 sectioning elements. We don’t want to rely on JS to fix old browser’s display problems.
  • We’re aiming for WCAG 2 compliance wherever realistic. Check the WCAG 2 Techniques before deciding whether or not the guideline is relevent or remains best-practice.
  • Obey the CSS styleguide, make life easier for everyone. Formatting notes are in the screen.core.css file.

Designers

  • Base font size is 16px, leading is 22px. The font is Helvetica Neue. Heading styles are found below.

Headings

Headings must be in the correct order and must not skip levels WCAG 2 1.3.1, 2.4.10

For all pages except the homepage the page title is a heading level one (on this page “Styleguide”). For the homepage the site title (contained in #site_context and usually replaced with a logo) is the heading level one.

Headings are set in Arvo Regular, all-caps.

Inline quotes don't need quotation marks because the user agent adds them itself, as in this example: That's what she said yeah but is it?

Images by default are inline the bottom of the image aligns with the baseline of the text within which it sits.

Blockquote

Blockquotes are used whenever the quote is not inline, or requires attributing.

This is quoted text, it doesn't have speech marks in the text itself, those are added by the CSS.

Blockquotes can contain any markup as they are block-level elements, including multiple paragraphs.

The mark-up convention is derived from http://html5doctor.com/blockquote-q-cite/ and http://adactio.com/journal/4675/.

Matt Wilcox, from the article Styleguide

Blockquote's are good, but the cite attribute isn't human-accessible, which is why you always want to put a hyperlink in there if the quoted source is available online. It's also worth noting that the cite is in a footer element in HTML5 because a footer gives information about the enclosing tag.

Popups

Popups are elements that can be clicked and enlarged on the same page, they can be floated left and right, or go full-width. They are available in multiple sizes.
